MetadataSchema¶

MCM module
MetadataSchema extension¶
MetadataSchema/Get¶

Gets one or all MetadataSchemas in the database.

Authentication:
  • Logged in – You need to be logged in to use this feature
  • Read permission – you will get the MetadataSchemas you have permission to read
Parameters:

metadataSchemaGUID (optional) – the GUID of the schema to get
Returns:

A list of MetadataSchemas

If no metadataSchemaGUID is given, all MetadataSchemas which the user has rights to read are returned.

MetadataSchema/Create¶

Adds a new MetadataSchema to the database.

You need to be logged in to use this feature and the user must have administritive rights.

Authentication:
  • Logged in – You need to be logged in to use this feature
  • Manage permission – Requires the SystemPermissons.Manage permission
Parameters:
  • name – a new name for the new schema
  • schemaXml – the XML for the new schema
  • metadataSchemaGUID (optional) – the GUID for new schema
Returns:

The GUID of the created MetadataSchema

If no metadataSchemaGUID is given, CHAOS assigns a GUID to the new schema. The new GUID is returned as a result.

MetadataSchema/Update¶

Updates a MetadataSchema in the database.

Authentication:
  • Logged in – You need to be logged in to use this feature
  • Write permission – you need to have write-permissions to the MetadataSchema
Parameters:
  • metadataSchemaGUID – the GUID of the schema to update
  • name – a new (or the same) name for schema
  • schemaXml – a new (or the same) XML for the schema
Returns:

The GUID of the updated MetadataSchema
MetadataSchema/Delete¶

Deletes a MetadataSchema in the database.

Authentication:
  • Logged in – You need to be logged in to use this feature
  • Delete permission – you need to have permissions delete the MetadataSchema (MetadataSchemaPermission.Delete)
Parameters:

guid – the GUID of the schema to delete
Returns:

The number of affected rows in the database
